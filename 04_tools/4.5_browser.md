## 4.5 浏览器自动化与计算机操作

智能体不仅需要调用 API，还需要与图形界面交互。**浏览器自动化（Browser Automation）** 和 **计算机操作（Computer Use）** 让智能体能够像人类一样操控 GUI 界面，打破了传统 API 的限制。

### 4.5.1 为什么需要浏览器自动化

并非所有系统都开放 API。遗留系统（ERP/CRM）和现代 SaaS 往往只有 Web 界面，要实现自动化操作，必须让智能体具备操控浏览器的能力。

**典型应用场景**：
- **Web 抓数据**：从没有 API 的网站提取信息
- **遗留系统集成**：自动操作老旧的企业内部系统
- **端到端测试**：模拟用户行为进行 UI 测试
- **RPA 任务**：订票、购物、表单填写等重复性工作

### 4.5.2 自动化技术的三代演进

当前主流的两种范式有着根本不同的工作方式：

- **DOM-based（基于文档对象模型）**：通过解析网页的 HTML 结构，使用 CSS 选择器或 XPath 定位元素。这是 Selenium/Playwright 的工作方式，精确高效，但 **依赖页面结构**——一旦开发者修改了 HTML 布局、类名或 ID，脚本就会失效。

- **视觉感知（Vision-based）**：通过截取屏幕图像，让多模态模型“看”界面并理解其含义。模型像人类一样识别“登录按钮在哪里”，然后输出坐标和操作指令。这种方式 **不依赖代码结构**，能够适应界面变化，但速度较慢且消耗更多计算资源。

| 维度 | DOM-based | 视觉感知 |
|------|-----------|----------|
| **工作方式** | 解析 HTML 结构 | 分析屏幕截图 |
| **选择器** | CSS/XPath | 视觉坐标 (x, y) |
| **精确性** | 高 | 中 |
| **速度** | 快 | 较慢 |
| **通用性** | 仅 Web | 任意 GUI |
| **维护成本** | 页面变化需更新 | 自适应 |
| **成本** | 低 | 高（消耗 Token） |

### 4.5.3 DOM-based 自动化

DOM-based 自动化通过程序化方式操控浏览器，适合结构稳定的网页场景。以下是使用 Playwright 实现登录的示例：


**适用场景**：
- 内部系统自动化（页面结构可控）
- 需要高速批量操作
- 测试自动化

### 4.5.6 主流浏览器自动化框架对比

在选择浏览器自动化框架时，需要考虑延迟、可靠性、成本和易用性等多个维度。

#### 框架对比表

| 框架 | 工作原理 | 性能 | 学习曲线 | 生产就绪 | 特色 |
|------|---------|------|--------|--------|------|
| **Playwright** | CDP协议 + 无头浏览器 | 快速（200-500ms） | 中 | ✅ | 跨浏览器、稳定 |
| **Selenium** | WebDriver标准 | 中等（500-2000ms） | 低 | ✅ | 生态成熟、广泛支持 |
| **Puppeteer** | Chrome DevTools Protocol | 快速（200-500ms） | 中 | ✅ | Chrome专用、轻量 |
| **视觉感知(Vision)** | 图像识别 + LLM | 慢速（2-5s） | 高 | ⚠️ | 通用性强、自适应 |

#### Playwright vs Selenium 深度对比

**Playwright 优势**：
- **跨浏览器原生支持**：同一套代码支持 Chromium、Firefox、WebKit
- **更快的性能**：使用 Chrome DevTools Protocol，避免了网络开销
- **现代API**：支持异步/等待模式，更符合当代开发习惯

**Selenium 优势**：
- **生态成熟**：大量已有项目、社区支持
- **跨语言**：支持 Java、Python、Ruby、JavaScript 等
- **学习资源丰富**：文档、教程、Stack Overflow 回答众多

### 4.5.7 视觉定位 vs DOM定位的权衡

#### DOM 定位的最佳实践

DOM 定位快速且可靠，但需要应对页面结构变化：

```python
class RobustDOMLocator:
    """健壮的 DOM 定位器"""

    def __init__(self, page):
        self.page = page

    def find_by_multiple_strategies(self, target_name: str):
        """
        使用多种策略查找元素，降低脆弱性
        """
        strategies = [
            # 策略1：精确的ID查找（最可靠）
            f"#{target_name}",

            # 策略2：使用 aria-label（无障碍标签，往往更稳定）
            f"[aria-label='{target_name}']",

            # 策略3：使用 data-testid（开发者预留给测试的属性）
            f"[data-testid='{target_name}']",

            # 策略4：包含文本的查找（页面结构重构时仍可用）
            f"button:has-text('{target_name}')",

            # 策略5：使用部分CSS类名（避免完全依赖类名）
            f"[class*='{target_name.lower()}']",
        ]

        for selector in strategies:
            try:
                element = self.page.query_selector(selector)
                if element:
                    print(f"Found element using strategy: {selector}")
                    return element
            except Exception:
                continue

        raise RuntimeError(f"Could not locate {target_name} using any strategy")

    def click_with_retry(self, selector, max_attempts=3):
        """点击，带重试和降级"""
        for attempt in range(max_attempts):
            try:
                element = self.find_by_multiple_strategies(selector)
                element.click()
                return True
            except Exception as e:
                if attempt < max_attempts - 1:
                    time.sleep(0.5)
                else:
                    # 降级：如果 DOM 点击失败，改用坐标点击
                    print("DOM click failed, falling back to coordinate click")
                    return self._click_by_vision(selector)

    def _click_by_vision(self, target_name):
        """降级方案：基于视觉的点击"""
        # 截屏 + 识别 + 点击
        screenshot = self.page.screenshot()
        coords = self._identify_target(screenshot, target_name)
        if coords:
            self.page.mouse.click(coords["x"], coords["y"])
            return True
        return False
```

#### 视觉感知的实际应用

```python
class VisionBasedInteraction:
    """基于视觉的交互"""

    def __init__(self, model):
        """model: 多模态LLM（如Claude Vision）"""
        self.model = model

    def interact_with_screenshot(self, task: str, screenshot_bytes: bytes):
        """
        给定截图和任务，模型决定如何操作

        例子：
        - 任务："点击最左边的红色按钮"
        - 输入：截图二进制数据
        - 输出：{"action": "click", "x": 100, "y": 50}
        """

        # 将截图转换为base64
        import base64
        encoded = base64.b64encode(screenshot_bytes).decode()

        # 调用视觉LLM
        response = self.model.generate(
            prompt=f"""
            Look at this screenshot and perform the following task:
            {task}

            Respond with a JSON object containing:
            - action: "click", "type", "scroll", etc.
            - target: description of what you're clicking
            - x, y: pixel coordinates (if clicking)
            - text: text to type (if typing)

            Example response format:
            {{"action": "click", "target": "submit button", "x": 342, "y": 215}}
            """,
            image_data=encoded,
            image_type="image/png"
        )

        # 解析响应
        action = json.loads(response.content)
        return action

    def loop_until_task_complete(self, task: str, page, max_iterations: int = 10):
        """
        循环式交互：截屏→识别→操作→检查→重复
        """
        for iteration in range(max_iterations):
            # 截屏
            screenshot = page.screenshot()

            # 判断任务是否完成
            completion_check = self.model.generate(
                prompt=f"""
                Has the following task been completed?
                Task: {task}

                Respond with JSON: {{"completed": true/false, "reason": "..."}}
                """,
                image_data=base64.b64encode(screenshot).decode(),
                image_type="image/png"
            )

            check_result = json.loads(completion_check.content)
            if check_result["completed"]:
                return {"success": True, "iterations": iteration}

            # 未完成，执行下一步操作
            action = self.interact_with_screenshot(task, screenshot)
            self._execute_action(page, action)

        return {"success": False, "reason": "max_iterations_reached"}
```

浏览器自动化在生产环境中面临多个稳定性问题，以下是常见的陷阱和解决方案：

#### 问题一：页面加载不完整

#### 问题二：网络不稳定导致的随机失败

```python
class ResilientBrowserAgent:
    """具有恢复能力的浏览器智能体"""

    def __init__(self):
        self.max_retries = 3
        self.timeout = 30  # 秒

    def execute_with_resilience(self, task, page):
        """
        带重试、超时和降级的执行
        """
        for attempt in range(self.max_retries):
            try:
                result = self._execute_task(task, page, timeout=self.timeout)
                return {"success": True, "result": result, "attempts": attempt + 1}

            except TimeoutError:
                print(f"Timeout on attempt {attempt + 1}, retrying...")
                if attempt < self.max_retries - 1:
                    # 网络超时，重新导航
                    page.reload()
                    time.sleep(2 ** attempt)  # 指数退避
                else:
                    return {"success": False, "reason": "network_timeout"}

            except Exception as e:
                print(f"Error on attempt {attempt + 1}: {e}")
                if attempt == self.max_retries - 1:
                    # 最后一次失败，返回降级响应或报警
                    return {
                        "success": False,
                        "reason": str(e),
                        "fallback_action": "escalate_to_human"
                    }

    def _execute_task(self, task, page, timeout):
        """执行任务，设置超时"""
        # 这里简化处理，实际应该用上下文管理器
        import signal

        def timeout_handler(signum, frame):
            raise TimeoutError("Task execution exceeded timeout")

        signal.signal(signal.SIGALRM, timeout_handler)
        signal.alarm(timeout)

        try:
            # 执行实际任务
            result = self._do_execute(task, page)
            return result
        finally:
            signal.alarm(0)  # 取消闹钟
```

```python
class JavaScriptAwareNavigator:
    """处理 JavaScript 渲染的导航器"""

    def navigate_and_wait_for_content(self, page, url):
        """
        某些网站的内容通过 JavaScript 动态生成
        需要等待 JavaScript 执行完成

                if (window.appReady) {
                    resolve();
                } else {
                    window.addEventListener('appInitialized', resolve);
                }
            })
        """)

        return page
```

#### 问题四：反爬虫对策

许多网站部署了反爬虫机制。以下是应对策略：

```python
class AntiCrawlerResistant:
    """应对反爬虫机制的浏览器智能体"""

    def __init__(self):
        # 设置 User-Agent，模拟真实浏览器
        self.user_agent = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"

        # 添加随机延迟，避免请求过于规则
        self.min_delay = 1
        self.max_delay = 5

    def launch_browser_with_stealth(self):
        """启动隐身模式的浏览器"""
        options = {
            "user_agent": self.user_agent,
            "viewport": {"width": 1920, "height": 1080},
            # 禁用无头模式指示
            "args": [
                "--disable-blink-features=AutomationControlled",
                "--disable-dev-shm-usage"
            ]
        }
        return options

    def navigate_with_human_like_behavior(self, page, url):
        """
        模拟真人行为：随机延迟、随机滚动等
        """
        import random
        import time

        # 随机延迟
        delay = random.uniform(self.min_delay, self.max_delay)
        time.sleep(delay)

        # 导航
        page.goto(url)

        # 随机滚动（某些网站检测滚动行为）
        for _ in range(random.randint(2, 5)):
            scroll_height = page.evaluate("window.innerHeight")
            page.evaluate(f"window.scrollBy(0, {random.randint(100, 500)})")
            time.sleep(random.uniform(0.5, 2))

        return page

    def handle_cloudflare_protection(self, page):
        """处理 Cloudflare 等高级反爬虫"""
        # 方案1：等待 Cloudflare JavaScript 执行完毕
        try:
            page.wait_for_load_state("networkidle", timeout=10000)
        except:
            pass

        # 方案2：如果还是被拦截，可以：
        # - 使用真实浏览器池服务（如 Browserless）
        # - 使用反爬虫绕过服务（如 Bright Data）
        # - 合作获取 API 而非爬取

        return page
```

### 4.5.9 Computer Use 类方案的最新进展

随着多模态模型（视觉+文本）的进展，出现了“Computer Use”类方案，让LLM能够像人一样操作任意GUI界面。

#### Computer Use 架构

```mermaid
graph LR

    Perception --> Decision["决策<br/>执行什么操作"]
    Decision --> Action["行动<br/>点击、输入、滚动"]
```

#### 实现原理

```python
class ComputerUseAgent:
    """Computer Use 类智能体"""

    def __init__(self, model, browser):
        self.model = model  # 多模态LLM
        self.browser = browser

    def act(self, goal: str, max_steps: int = 10):
        """
        循环式执行，直到完成任务
        """
        history = []

        for step in range(max_steps):
            # 第1步：截屏
            screenshot = self.browser.screenshot()

            # 第2步：让模型分析截图和目标，决定下一步操作
            response = self.model.generate(
                messages=[
                    {
                        "role": "user",
                        "content": [
                            {
                                "type": "image",
                                "source": {"type": "base64", "media_type": "image/png", "data": screenshot}
                            },
                            {
                                "type": "text",
                                "text": self._build_prompt(goal, history, step)
                            }
                        ]
                    }
                ],
                system_prompt=self._get_system_prompt()
            )

            # 第3步：解析模型的操作指令
            action = self._parse_action(response.content)
            history.append({"screenshot": screenshot, "action": action})

            # 第4步：执行操作
            if action["type"] == "finish":
                return {"success": True, "result": action.get("result"), "steps": step}

            self._execute_action(action)

            # 第5步：等待、观察效果
            time.sleep(1)

        return {"success": False, "reason": "max_steps_exceeded"}

    def _build_prompt(self, goal: str, history: list, step: int):
        """构造提示词"""
        context = f"""
You are operating a computer to achieve the following goal:
{goal}

Current step: {step}

{self._format_history(history)}

Based on the current screenshot above, what is your next action?
Respond in JSON format:
{{
    "type": "click" | "type" | "scroll" | "finish",
    "target": "description of what you're interacting with",
    "x": 100,  // for click
    "y": 200,  // for click
    "text": "text to type",  // for type
    "result": "final answer"  // for finish
}}
        """
        return context

    def _execute_action(self, action: dict):
        """执行操作"""
        if action["type"] == "click":
            self.browser.click(action["x"], action["y"])
        elif action["type"] == "type":
            self.browser.type(action["text"])
        elif action["type"] == "scroll":
            self.browser.scroll(action.get("direction", "down"))
```

#### Computer Use 的优势与局限

| 维度 | 优势 | 局限 |
|------|------|------|
| **通用性** | 可操作任意GUI，无需集成 API | 速度慢（每次 2-5秒） |
| **鲁棒性** | 页面结构变化不影响操作 | 成本高（每次视觉分析）|
| **易用性** | 开发简单，快速原型 | 容易被反爬虫阻止 |

### 4.5.10 实战建议：选择合适的方案

```
任务类型与方案选择决策树：

├─ 内部系统/已有API
│  └─ 使用 API 或数据库连接（最佳）
│
├─ 稳定的商业网站（如 AWS、GitHub）
│  └─ 使用 DOM-based（Playwright/Selenium）+ XPath
│
├─ 动态渲染+频繁变化的网站
│  └─ 使用视觉感知（Vision + Computer Use）
│
├─ 反爬虫保护很强的网站
│  └─ 考虑：(1) 是否有 API，(2) 是否值得成本，(3) 人工操作
│
└─ 一次性/原型任务
   └─ 使用 Computer Use（快速开发）
```

#### 安全配置示例

以下示例展示了如何实现操作黑名单、高风险确认和审计日志：

```python
class SecureComputerUse:
    """安全的计算机操作封装"""
    
    BLOCKED_ACTIONS = [
        ("navigate", r".*bank.*"),      # 禁止访问银行网站
        ("type", r"password=.*"),       # 禁止输入密码
        ("click", r".*delete.*all.*"),  # 禁止点击全删除
    ]
    
    HIGH_RISK_PATTERNS = [
        r".*payment.*",
        r".*checkout.*",
        r".*confirm.*order.*",
    ]
    
    def execute(self, action: dict) -> dict:
        # 1. 检查黑名单

        for action_type, pattern in self.BLOCKED_ACTIONS:
            if action["type"] == action_type:
                if re.match(pattern, str(action.get("target", ""))):
                    return {"blocked": True, "reason": "Forbidden action"}
        
        # 2. 高风险操作需确认

        if any(re.match(p, str(action)) for p in self.HIGH_RISK_PATTERNS):
            if not self.get_user_confirmation(action):
                return {"blocked": True, "reason": "User declined"}
        
        # 3. 记录审计日志

        self.audit_log(action)
        
        # 4. 执行操作

        return self.do_execute(action)
```
