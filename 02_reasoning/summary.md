## 本章小结

本章深入探讨了智能体认知架构的三大支柱：规划、记忆与行动，揭示了让 Agent 具备“智慧”的微观机制。

### 核心要点回顾

1.  **规划与决策 (Planning)**
    *   **任务分解**：利用 Chain of Thought (CoT) 和 Tree of Thoughts (ToT) 将复杂目标拆解为可执行步骤。
    *   **核心模式**：
        *   **ReAct**：推理与行动交替进行，适合大多数通用任务。
        *   **Plan-and-Solve**：先制定完整计划再执行，宏观把控更强。
    *   **反思 (Reflection)**：通过分析错误日志进行自我批评与修正 (Reflexion)，实现自我进化。

2.  **记忆系统 (Memory)**
    *   **多级记忆**：区分感觉记忆、短期记忆（Context Window）与长期记忆。
    *   **RAG 技术**：利用向量嵌入 (Embeddings) 和向量数据库，实现基于语义的知识检索与增强。
    *   **高级策略**：记忆合成、分层索引与遗忘机制，防止记忆臃肿并提高检索准确性。

3.  **行动系统 (Tools)**
    *   **Function Calling**：将自然语言转换为结构化的 API 调用请求 (JSON)，连接 LLM 与外部世界。
    *   **工具设计**：清晰的 Schema 定义与文档字符串 (Docstrings) 是 Agent 正确使用工具的关键。
    *   **鲁棒性**：参数校验与错误处理机制确保行动的安全与稳定。
