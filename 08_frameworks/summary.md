## 本章小结

本章全面介绍了智能体开发框架生态，从开源框架到平台级产品，帮助开发者根据场景选择合适的工具。

### 核心要点回顾

1.  **框架生态概览**
    *   **LangGraph**：**智能体即图 (Agent as a Graph)**。强调深度控制与状态管理，适合构建复杂的、生产级的业务流。
    *   **AutoGen**：**通过对话校准 (Calibration through Conversation)**。强调多智能体对话与代码执行，是编程智能体 (Coding Agent) 和科研探索的首选。
    *   **CrewAI**：**角色扮演小队 (Role-Playing Squad)**。强调角色扮演与易用性，适合内容创作和快速原型开发。

2.  **LangGraph 实战：深度控制**
    *   利用 **状态图 (StateGraph)** 构建状态机，清晰定义节点 (Node) 与边 (Edge)。
    *   通过 **检查点 (Checkpointer)** 实现状态持久化，天然支持 **Human-in-the-loop**（人工审批/断点续传），为企业级应用提供了必要的确定性。

3.  **LlamaIndex：数据驱动的 RAG 专家**
    *   专注于数据索引与检索，在 RAG 场景中表现出色。
    *   通过 Sub-Question 引擎和多数据源集成，实现复杂数据分析。

4.  **企业级集成与平台产品**
    *   **Semantic Kernel**：多语言支持（C#/Python/Java），适合企业现有系统集成。
    *   **OpenAI Assistants API**：开箱即用的智能体构建平台，内置代码解释器和文件搜索。
    *   **Claude Tool Use**：Anthropic 的工具调用能力，支持计算机使用等创新功能。

5.  **性能基准评测**
    *   延迟、内存、Token 成本和并发能力的量化对比，为框架选型提供数据支撑。

| 场景 | 推荐框架 | 核心理由 |
| :--- | :--- | :--- |
| **企业核心业务流** | LangGraph | 状态可控、支持人工介入、容错强 |
| **代码生成/数据分析** | AutoGen | 内置代码解释器、Docker 沙箱支持 |
| **内容创作/快速 Demo** | CrewAI | 上手快、角色定义清晰、心智负担小 |
| **数据检索/RAG 应用** | LlamaIndex | 数据索引能力强、多格式支持 |
| **企业现有系统集成** | Semantic Kernel | 多语言支持、企业级架构 |
